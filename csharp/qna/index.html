Q1. Can we overload the Main() method in C#?
Yes, we can overload the main method in the C# programming language. A class can have any number of main methods, but to run, we must have the main method or main method with signature as public static void Main(string[] args). Important Note: We can overload the main method, but it can't be overridden because it is the static method. Also, the static method can't be virtual or abstract. The following example shows how to overload the main method in C#.
{{ site.csfile }}
{% highlight csharp %}
using System;
namespace Kodingwindow
{
    class Program
    {
        public static void Main(string s) //Not an entry point
        {
            Console.WriteLine("Overloaded "+s);
        }
        //public static void Main() //Entry point
        public static void Main(string[] args) //Entry point
        {
            Console.WriteLine("Main method");
            Main("Main method");
            Console.ReadKey();
        }
    }
}
{% endhighlight %}

Overloaded Main method
Q2. Does C# support multiple inheritance?
The answer to this question is no, but you can say that the multiple inheritance can be achieved in C# using the interface concept. The following example illustrates how to achieve it.
{{ site.csfile }}
{% highlight csharp %}
using System;
namespace Kodingwindow
{
    interface YesBank
    {
        void interest();
    }
    interface NoBank
    {
        void interest();
    }
    class Bank: YesBank, NoBank
    {
        void YesBank.interest()
        {
            Console.WriteLine("Interest Rate: 10%");
        }
        void NoBank.interest()
        {
            Console.WriteLine("Interest Rate: 8%");
        }
    }
    class Program
    {
        public static void Main(string[] args)
        {
            YesBank y = new Bank();
            y.interest();

            NoBank n = new Bank();
            n.interest();
            Console.ReadKey();
        }
    }
}
{% endhighlight %}
{{ site.output }}
Interest Rate: 10%
Interest Rate: 8%
Q3. What is the difference between ref and out parameters?
We use ref and out parameters when we want to pass parameters to the methods like passing parameters using reference and output. The difference between those two is that an argument passed as ref must be initialized before passing to the method, whereas out parameter not initialized before passing to a method.

Q5. What is Generic Collections and enlist few?
Generic collections are strongly type-safe, which means the elements we are going to add in the collection must be of the same type. The generic collections are List, Dictionary, HashSet, Stack, etc. For practical examples, refer to C# Collections
Q6. Which methods used to remove the element from Stack and Queue?
The Pop() method used to remove the top element from the Stack, whereas to remove the front element from the Queue, we use Dequeue() method. For practical examples, refer to C# Collections
Q7. What is the difference between ArrayList and List?
The ArrayList class is a non-generic collection resides in System.Collection namespace, whereas the List class is generic collection resides in System.Collection.Generic namespace. It is recommended to use the generic collection as it stores the same type of values (List<T> where T is the type of value). For practical examples, refer to C# Collections
Q8. What is boxing and unboxing in C#?
In boxing, we convert a value type to object type. Whereas in unboxing, we convert an object type to a value type.

One can have nested namespaces too, but it depends on the project requirements and the approach you follow. For practical examples, refer to C# Namespaces
Q10. Can we have try block only without having catch and finally block?
Answer to this question is no one can’t have only try block. It must be followed by catch block or a finally block. The try block followed by a catch or multiple catch blocks are fine, moreover, try followed by finally block is fine too.
{{ site.csfile }}
{% highlight csharp %}
using System;
namespace Kodingwindow
{
    class Program
    {
        public static void Main(string[] args)
        {
            try
            {
                Console.WriteLine("Hello, World");
            }
            finally
            {
                Console.WriteLine("Finally block always get executed");
            }
            Console.ReadKey();
        }
    }
}

{% endhighlight %}
{{ site.output }}
Hello, World 
Finally block always get executed
Q11. Can multiple catch blocks be executed?
The answer to this question is no multiple catch blocks can't be executed. When one catch block gets executed, then control skips all other catch blocks.
Q12. What is a delegate and multicast delegate in C#?
A delegate is an object that holds a reference to a method. This concept is similar to function pointers in C/C++ programming language. A delegate holds references only to those methods whose signature (parameter list) matches with the delegate. Whereas the multicast delegate holds more than one reference of methods at a time.
Q13. What is thread synchronization in C#?
When multiple threads accessing the same resources and create a concurrency problem, then we use C# thread synchronization to synchronize access to a shared resource or coordinate thread interaction. The synchronization in C# achieved using the lock keyword. The lock controls access to block of code with an object.
